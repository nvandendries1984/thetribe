Introduction

Welcome to the TheTribe Discord Bot REST API. This API allows you to interact with the bot programmatically to retrieve information about guilds, members, moderation logs, and more.

Base URL

http://188.212.125.248:15016 or http://thetribe.bytebees.dev:15016

Response Format

All API responses are in JSON format. Error responses include:

{
  "error": true,
  "message": "Error description",
  "code": 404
}

Get detailed information about a specific guild.

guildId - Discord Guild ID

GET /api/guilds/{guildId}/members

Get list of members in a guild.

Parameters

guildId - Discord Guild ID

Query Parameters

limit - Number of members to return (default: 50, max: 1000)
after - User ID to start after (for pagination)
GET /api/guilds/{guildId}/members/{userId}

Get detailed information about a specific guild member.

Parameters

guildId - Discord Guild ID
userId - Discord User ID

Moderation

GET /api/warnings

Get list of all warnings across all guilds.

Query Parameters

guild - Filter by Guild ID
user - Filter by User ID
limit - Number of warnings to return (default: 50)
skip - Number of warnings to skip (pagination)
GET /api/warnings/{userId}

Get all warnings for a specific user.

Parameters

userId - Discord User ID

Query Parameters

guild - Filter by Guild ID
GET /api/modlogs

Get moderation logs.

Query Parameters

guild - Filter by Guild ID
action - Filter by action type (ban, kick, timeout, warn)
moderator - Filter by Moderator ID
limit - Number of logs to return (default: 50)
skip - Number of logs to skip (pagination)

Authentication & Limits

Authentication

Currently, this API does not require authentication. In production environments, consider implementing API keys or OAuth2 for security.

Rate Limiting

Currently no rate limiting is implemented. For production use, consider implementing rate limiting to prevent abuse.

CORS

CORS is enabled for all origins (*). This allows web applications to make requests to the API from any domain.
